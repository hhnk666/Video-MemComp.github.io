🔍 Overview

Video-MemComp is a structure-aware KV memory compression framework derived from a first-order Taylor expansion of error propagation. It effectively decomposes compression error into Local Approximation Error and Layer-wise Structural Sensitivity.

Figure 2. Overview of Video-MemComp Framework: guided by Taylor expansion, we minimize local error via Pre-RoPE aggregation and manage sensitivity via a Tri-Hybrid budget allocation.

Strict O(1) Memory

Fixed KV memory usage regardless of video length.

Zero Offloading

No CPU/GPU data transfer latency during inference.

📊 Experimental Results

1. Comprehensive Offline Video Understanding

We evaluate Video-MemComp across three representative offline benchmarks: VideoMME (long-term), MLVU (multi-task), and EgoSchema (egocentric). As shown in Table 1, our method consistently meets or even exceeds the performance of full-cache baselines while using significantly less memory. This indicates that our Taylor expansion-guided strategy effectively identifies and preserves critical semantic information while discarding temporal redundancy.

Table 1. Performance Comparison on Offline Video Understanding Benchmarks. Bold and underlined values mark the best and second-best results.

2. Robustness Across Memory Budgets

A key advantage of Video-MemComp is its stability under various memory constraints. Figure 3 illustrates the performance curve on MLVU using the Qwen2-VL-7B backbone. Our method outpaces all strong baselines (like SnapKV and InfiniPot) across the entire budget spectrum. Notably, at 6K, 12K, and 24K budgets, Video-MemComp surpasses the Full-KV performance, demonstrating that "more memory" isn't always better—targeted compression can actually reduce "attention noise" in long videos.

Figure 3. Accuracy vs. KV Cache Budget on MLVU. Our method maintains a superior accuracy-memory trade-off.

3. Streaming Understanding & System Efficiency

In real-world streaming scenarios (RVS Benchmarks), Video-MemComp achieves the highest average accuracy among all memory-constrained methods. Unlike retrieval-based approaches (e.g., ReKV) that suffer from high latency, our Taylor-expansion framework provides a direct, training-free compression path.

Hardware efficiency results in Table 3 show that our framework reduces Peak Memory by ~27% and Time-to-First-Token (TTFT) by ~50% compared to ReKV. This efficiency enables long-video inference on consumer-grade GPUs (like RTX 4090) that would otherwise trigger Out-of-Memory (OOM) errors.
